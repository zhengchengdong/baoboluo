## 1. 游戏概述

两名玩家对抗：
- **进攻方**：逐根绑皮筋到菠萝上，皮筋收缩挤压菠萝，直到挤爆
- **防守方**：割断皮筋释放压力，延缓爆炸

核心视觉目标：
- 皮筋勒入处形成**平滑弧形凹陷**（非矩形平底）
- 皮筋挤压 → 体积守恒 → 未受约束区域（两端）自然鼓出
- 皮筋越多、越紧 → 应力累积 → 爆炸

## 2. 为什么选 PBD（Position-Based Dynamics）

### 2.1 核心思路

PBD 不显式计算力。而是定义**约束**（目标条件），每帧反复修正顶点位置来满足所有约束。约束之间互相“拉扯” → 自然收敛到力平衡态。

### 2.2 类比

> 两个人抢一个东西——你不需要知道他们各使多大劲，看最后东西停在哪就知道了。停在哪儿，就是力平衡。

### 2.3 关键公式

每个约束的修正公式：

```
Δx = (目标值 − 当前值) × 刚度
```

刚度 < 1（通常 0.3~0.8）。**每次只朝目标挪一小步**，不能一步到位。

为什么必须刚度 < 1：
- 刚度 = 1.0 → 每轮都“达标才停” → 幂等 → 永远振荡，永不收敛
- 刚度 < 1.0 → 每轮只修正误差的一部分 → 误差逐轮缩小 → 收敛

## 3. 两条核心约束

### 3.1 皮筋约束（Band Constraint）

**目标**：皮筋所在高度的截面半径被压缩到目标值。

**原理**：
- 皮筋 = 一条环形约束，作用于菠萝表面某个高度附近
- 影响范围由核函数（smoothstep）控制：中心最强，边缘为 0
- 有效目标半径随距离渐变（**这是避免矩形凹槽的关键**）：

```
dy = |顶点Y − 皮筋中心Y|
t = dy / influenceRadius        (0 = 中心, 1 = 边缘)
effectiveTarget = tightR + (当前r − tightR) × smoothstep(0, 1, t)

dy = 0（中心）→ target = tightR           ← 最紧
dy = influence（边缘）→ target = 当前r    ← 不动
```

这样凹陷自然形成 U 形弧线 `╲___╱`，而非矩形平底 `████`。

### 3.2 体积约束（Volume Constraint）

**目标**：菠萝总体积保持不变。

**原理**：
- 遍历所有三角形面，累加计算当前体积
- 若 $V_{current} < V_{rest}$（被皮筋压小了）：
  - 所有顶点沿径向（远离 Y 轴方向）推出
  - 推量 $\propto \frac{V_{rest} - V_{current}}{\text{总表面积}}$

**为什么两端会鼓出**：
- 皮筋约束是**局部的**（只管那一圈）
- 体积约束是**全局的**（管整个菠萝）
- 皮筋区：皮筋往里推 vs 体积往外推 → 妥协 → 凹进去但不太深
- 自由区（两端）：只有体积往外推 → 单方面累积 → 鼓出

> 就像捏气球——手指捏进去的地方凹了，别的地方鼓出来补偿。总体积不变。

### 3.3 两条约束如何决定“勒多深”

```
每帧迭代 5~10 次：

  第 1 步：皮筋约束 → 接触区顶点往内推
  第 2 步：体积约束 → 所有顶点往外推

→ 皮筋约束把顶点推进去
→ 体积约束又推出来一些
→ 下一轮皮筋又推进去
→ ……
→ 收敛 = 力平衡 = “勒多深”的物理答案
```

**刚度比**控制平衡点：

| 皮筋刚度 | 体积刚度 | 结果 |
|---------|---------|------|
| 大 | 小 | 勒得深，体积被压缩 |
| 小 | 大 | 勒得浅，体积几乎不变，两端鼓很多 |

## 4. 多根皮筋的叠加效应

### 4.1 为什么皮筋越多勒得越深

PBD 每轮迭代遍历所有约束。两根皮筋作用在相邻区域：

```
约束列表 = [皮筋A, 体积, 皮筋B]
（每轮迭代顺序）

皮筋A 往里推 → 体积往外推（A 松了一点）
→ 皮筋B 往里推（B 替 A 补了位置）
→ 净向内 > 净向外 → 越来越深
```

**本质**：多条皮筋 → 向内修正次数多 → 投票占优 → 勒得更深。

### 4.2 皮筋间距的影响

- **间距很近**：核函数重叠，互相“掏地基”，每条都比单独时深
- **间距远**：各自独立，体积约束的全局反抗占主导 → 相互变浅
- **临界间距**：$d > \sigma/\sqrt{2}$（$\sigma$ = 核函数宽度）

### 4.3 正态分布 → 自然弧度

玩家手抖，皮筋集中在腰部（正态分布）：

```
边缘     中间     边缘
 1次    3次    5次    3次    1次   (每轮约束执行次数)
 浅     较深    最深   较深    浅    (凹入深度)

→ 自然形成中间深、边缘浅的弧线
→ 不需要手动设计弧度
```

## 5. 关键参数

| 参数 | 含义 | 建议值 |
|------|------|--------|
| `BAND_STIFFNESS` | 皮筋刚度（越大越硬） | 0.4~0.65 |
| `BAND_INFLUENCE` | 皮筋影响范围（世界单位） | 0.3~0.5 |
| `BAND_INITIAL_TIGHTNESS` | 绑上去时的紧度（%当前半径） | 0.75~0.85 |
| `VOLUME_STIFFNESS` | 体积保持刚度 | 0.3~0.6 |
| `PBD_ITERATIONS` | 每帧迭代次数 | 5~10 |
| `STRESS_EXPLODE` | 爆炸压力阈值（%） | 85~95 |

## 6. 性能与代理网格

### 6.1 问题

高精度模型（~200万面）直接做 PBD → 体积计算遍历所有面 → 每帧 ~3 FPS。

### 6.2 方案：代理网格（Proxy Mesh）

```
高模（~10万面）：  只用来看，不参与物理
低模（~3000面）：  只参与物理，不给玩家看
```

加载时建立**重心坐标映射**：高模每个顶点 → 低模哪个三角形 + 重心坐标。运行时低模 PBD 变形 → 映射驱动高模渲染。

### 6.3 性能估算

| 模型面数 | 体积计算/迭代 | 预计帧率 |
|---------|-------------|---------|
| 194万（原模型） | ~50ms | ~3 FPS |
| 5000（精简后） | ~0.1ms | 60 FPS |

## 7. 爆炸机制（待实现）

```
压力计算 = 体积损失比例 × 100 + 皮筋数量加成

压力 ≥ 阈值 → 触发爆炸：
  1. 隐藏菠萝模型
  2. 从表面采样顶点，生成粒子
  3. 粒子沿法线 + 随机方向飞出
  4. 加重力下落
  5. 数秒后消失
```

## 8. 约束执行顺序（每帧伪代码）

```javascript
每帧:
  // PBD 迭代
  for iter in 1..PBD_ITERATIONS:
    // 1. 皮筋约束
    for each band:
      for each vertex within influence range:
        effectiveTarget = lerp(tightR, currentR, smoothstep(distance))
        if r > effectiveTarget:
          push radially inward toward effectiveTarget

    // 2. 体积约束
    V_current = computeVolume(all triangles)
    deficit = V_rest - V_current
    if deficit > 0:
      push ALL vertices radially outward

  // 更新渲染
  computeVertexNormals()
  updateBandVisuals()
  computeStress()
  if stress > threshold: explode()
```
