## 1. 游戏概述

两名玩家对抗：
- **进攻方**：点击菠萝表面逐根绑皮筋，皮筋收缩挤压，直到菠萝爆裂
- **防守方**：割断皮筋释放压力，延缓爆炸

## 2. PBD 核心原理

### 2.1 基本概念

PBD 不显式计算力。定义**约束**（目标条件），每帧反复修正顶点位置来满足它们。

每条约束的修正公式：
```
Δp = (目标值 − 当前值) × 刚度
```
刚度 < 1，每次只朝目标挪一小步。多轮迭代收敛到平衡态。

### 2.2 为什么刚度 < 1

- 刚度 = 1.0 → 每轮"一步到位" → 幂等 → 永远振荡，不收敛
- 刚度 < 1.0 → 每轮只修正误差的一部分 → 误差逐轮缩小 → 收敛

### 2.3 多约束竞争

一个顶点被多条约束同时管理。每轮迭代遍历所有约束，各自修正一次：
- 约束推同一个顶点 → 方向相同则叠加，方向相反则抵消
- 刚度比决定最终平衡点（刚度大的约束"赢"得多）

## 3. 三条约束

| # | 约束 | 作用 | 刚度含义 |
|---|------|------|---------|
| ① | **形状约束** | 每个顶点拉回初始位置 | 菠萝材料弹性（高=硬） |
| ② | **皮筋约束** | 皮筋接触区径向压缩 | 皮筋松紧度（高=紧） |
| ③ | **体积约束** | 总体积保持不变 | 不可压缩性（高=刚性） |

### 3.1 形状约束

```
Δp_i = SHAPE_STIFFNESS × (p_rest[i] − p_current[i])
```

作用于**全部顶点**。对抗皮筋约束的"对手"。刚度越高，菠萝越硬，皮筋越勒不进去。

### 3.2 皮筋约束

作用于皮筋高度 ± 影响半径内的顶点。有效目标半径随距离渐变（避免矩形平底）：

```
dy = |顶点Y − 皮筋中心Y|
t = dy / influenceRadius         (0=中心, 1=边缘)
effectiveTarget = tightR + (当前r − tightR) × smoothstep(0, 1, t)

若 r > effectiveTarget:  径向缩放顶点到 effectiveTarget
```

### 3.3 体积约束

遍历所有三角形面计算当前体积 V_current。若 V_current < V_rest（被压小了）：

所有顶点沿径向（远离 Y 轴）均匀外推，推量 ∝ (VOLUME_STIFFNESS × deficit) / 总表面积。

### 3.4 三条约束如何博弈

每帧迭代 N 次，约束执行顺序：

```
① 形状约束: 所有顶点 → rest
② 皮筋约束: 接触区顶点 → targetRadius
③ 体积约束: 所有顶点 → 均匀外推
```

**皮筋区顶点（正面交锋）**：
- 形状往外拉 ←→ 皮筋往里推
- 体积也往外推（但它均匀分布，贡献较小）
- **形状刚度 vs 皮筋刚度 → 决定勒多深**

**自由区顶点（无皮筋）**：
- 形状约束想拉回原位
- 体积约束想往外推
- 体积刚度 > 形状刚度 → 两端鼓出

**割断皮筋后**：
- 只剩形状约束 → 顶点弹回原位 → 椭球恢复

## 4. 多根皮筋叠加

### 4.1 间距近 → 互相加深

两根皮筋相邻时，中间区域顶点被两边同时往内推 → 每根勒得比单独时更深。

### 4.2 间距远 → 互相抵消

各自独立，体积约束全局反抗占主导 → 每根变浅。

### 4.3 正态分布 → 自然弧度

皮筋集中在腰部（正态分布密度）→ 中间约束执行次数多 → 自动形成中间深、边缘浅的弧线。

## 5. 参数表

| 参数 | 含义 | 默认值 |
|------|------|--------|
| `SHAPE_STIFFNESS` | 菠萝材料弹性 | 0.30 |
| `BAND_STIFFNESS` | 皮筋松紧度 | 0.50 |
| `VOLUME_STIFFNESS` | 体积保持力度 | 0.40 |
| `BAND_INFLUENCE` | 皮筋影响范围（核函数宽度） | 0.60 |
| `BAND_INITIAL_TIGHTNESS` | 绑上时目标半径 = 当前×此值 | 0.80 |
| `PBD_ITERATIONS` | 每帧迭代次数 | 5~10 |

## 6. 约束伪代码

```javascript
每帧:
  if bands.length > 0:
    for iter in 1..PBD_ITERATIONS:
      pbdApplyShape(posArr)            // ① 全部顶点 → rest
      for band in bands:
        pbdApplyBand(band, posArr)     // ② 局部径向压缩
      pbdApplyVolume(posArr, idxArr)   // ③ 均匀外推补体积
  else:
    for iter in 1..PBD_ITERATIONS:
      pbdApplyShape(posArr)            // 无皮筋，恢复原状

  更新法线、sprite、band视觉
```

## 7. 模型与性能

- 调试阶段使用低面数椭球（42 顶点），60 FPS，便于验证 PBD 逻辑
- 正式阶段使用精简后的代理网格（~3000 面）做物理，高模（~10 万面）做渲染，顶点映射
- 原始菠萝模型 194 万面直接跑 PBD ≈ 3 FPS，不可用
